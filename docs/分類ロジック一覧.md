# 技術分類ロジック 一覧（特許・論文）

技術ポートフォリオの大分類・小分類が、どのデータをどう選別して集計されているかの早見表です。

---

## 1. 全体像

| 階層 | 特許の判定材料 | 論文の判定材料 |
|------|--------------|--------------|
| **大分類**（9分類） | 特許の **IPCコード** | OpenAlexの **subfield**（研究小領域ラベル） |
| **小分類**（各6＋汎用1） | **IPCコード**＋タイトル/要約の**キーワード** | OpenAlexの **トピック名**（display_name） |

- 大分類と小分類は**別々の材料で独立に判定**します（同じ大分類でも小分類の付き方は文献ごとに異なる）。
- 1件が複数の小分類に該当することがあります（非排他）。
- どの小分類にも当たらない文献は、その大分類の受け皿「**汎用・基盤技術**」に入ります。

---

## 2. 大分類の選別基準

| | 特許 | 論文 |
|---|------|------|
| **入力データ** | `patents.ipc`（IPC分類コード） | `works.topics[].subfield` |
| **対応表** | `taxonomy_patent_class`（29ルール） | `paper_subfield_taxonomy` |
| **照合方法** | IPCコードの完全一致／前方一致 | subfield名の完全一致 |
| **集計単位** | 特許番号のユニーク件数（企業×大分類×年） | 論文IDのユニーク件数（企業×大分類×年） |

**大分類（9カテゴリ）:** 次世代通信 / AI・認識 / コンピューティング基盤 / 蓄電・電力 / 映像・ディスプレイ / 半導体デバイス / ヘルスケア・医薬 / 材料・高分子 / センシング・測位

---

## 3. 小分類の選別基準

### 特許の小分類

| 判定材料 | 対応表・方式 | 例 |
|---------|------------|-----|
| **IPCコード** | `taxonomy_patent_keyword`（match_mode=`ipc`／前方一致） | `H01M`→リチウムイオン電池、`H04N`→映像符号化、`H04W/H04L`→ネットワーク基盤 |
| **キーワード（単語）** | 同表（match_mode=`word`／単語境界一致） | `mimo`→Massive MIMO、`transistor`→トランジスタ |
| **キーワード（連語）** | 同表（match_mode=`phrase`／部分一致） | `cloud computing`→クラウド、`silicon carbide`→パワー半導体 |

- 照合対象テキスト：`title_en`＋`title_ja`＋`abstract_epo`
- **親大分類に属する特許のみ**、その配下の小分類を判定（誤爆防止）

### 論文の小分類

| 判定材料 | 対応表・方式 | 例 |
|---------|------------|-----|
| **トピック名** | `taxonomy_paper_topic_pattern`→`taxonomy_paper_topic_map`（ILIKE 部分一致） | 「Cloud Computing…」→クラウド、「Cancer…」→創薬・医薬品 |

- 入力：OpenAlexの `topics[].display_name`（構造化された研究トピックラベル）
- テキスト照合ではなく**トピック名で直接マッチング**するため精度が高い
- **親大分類に属する論文のみ**、その配下の小分類を判定

## 5. 集計上の注意（説明時のポイント）

- **非排他集計：** 1件が複数の小分類に該当しうるため、小分類の単純合計は大分類を上回ることがあります。
- **親子は独立判定：** 大分類はコード（IPC/subfield）、小分類はキーワード/トピック名で判定。抽出方法が異なるため、両者は完全一致しません。
- **カウント方法：** いずれも `count(DISTINCT 特許番号 / 論文ID)` によるユニーク件数。
- **データ更新時：** ルール（対応表）は永続化済み。新データ取り込み後は `refresh_all_signals.sql` を実行すれば全集計が最新化されます。
